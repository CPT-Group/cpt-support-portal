## 🎯 **Architectural Opinions**

### 1. **SVG vs. Mermaid (Confidence: 0.92)**

**Recommendation:** Use Mermaid for 95% of documentation diagrams, SVG for complex custom layouts.

**Decision Factors:**
- Standard diagram types → Mermaid (flowchart, sequence, class, ERD, state)
- Complex custom layouts → SVG (pixel-perfect positioning, advanced styling)
- **ASCII art → NEVER (forbidden, confidence 0.0)**

**Rationale:** Mermaid provides text-based syntax with professional rendering, is faster to create, and works across all documentation platforms. SVG is necessary when Mermaid has limitations.

---

### 2. **Icon Library Selection (Confidence: 0.90)**

**Recommendation:** 
- **Primary:** Bootstrap Icons (2,000+, MIT)
- **Secondary:** Iconoir (1,671, MIT, minimalist)
- **Tertiary:** SVG Repo (500,000+, specialized needs)
- **Flowcharts:** Flaticon (diagram-specific symbols)

**Decision Factors:**
- Technical diagrams → Bootstrap Icons
- Modern clean aesthetic → Iconoir
- Specialized/unique needs → SVG Repo
- Flowchart symbols → Flaticon

**Rationale:** Bootstrap Icons provides consistent design, comprehensive technical coverage, and MIT licensing, making it ideal for 80% of documentation diagram needs.

---

### 3. **Integration Method (Confidence: 0.88)**

**Recommendation:**
- **Repeated icons:** SVG sprite sheets (define once, use many times)
- **Unique diagrams:** Inline SVG (full control, no HTTP requests)
- **Large standalone:** External SVG files (cacheable)

**Decision Factors:**
- Reuse frequency (multiple → Sprite; single → Inline; cross-page → External)
- Diagram size (small → Inline; large → External)
- Performance priority (repeated → Sprite for optimization)

**Rationale:** Sprite sheets optimize performance for repeated icons, inline SVG works best for unique diagrams, external files suit large standalone diagrams.

## 🚫 **CRITICAL: ASCII Art Prohibition**

### Absolute Rule (Confidence: 1.0)

**ASCII art diagrams are STRICTLY FORBIDDEN in all professional documentation.**

**Why Forbidden:**
1. Unprofessional appearance (looks like 1980s terminal)
2. Unreadable (breaks on different fonts, terminals, editors)
3. Inconsistent rendering (looks different everywhere)
4. Not maintainable (nightmare to edit, align, update)
5. Accessibility nightmare (screen readers fail)
6. No styling options (monochrome, fixed size, ugly)
7. User frustration (as evidenced by feedback)

**MANDATORY Alternatives:**
- ✅ **Mermaid** (95% of diagrams) - text-based, renders professionally
- ✅ **SVG** (5% of diagrams) - complex custom layouts

**This prohibition has ZERO exceptions. ASCII art confidence level: 0.0 (forbidden).**
